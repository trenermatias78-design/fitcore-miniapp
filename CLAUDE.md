# FitCore Mini App — Frontend

## Стек
- **React 18 + Vite** — SPA, деплой на Railway (`VITE_API_URL` → `api.py`)
- **Один файл:** `src/FitCoreApp.jsx` (~5000+ рядків) — всі компоненти, екрани, стилі
- **Telegram WebApp SDK** — `window.Telegram.WebApp`, `initData` для авторизації, haptic feedback
- Стилізація: CSS-in-JS (inline styles) + глобальний `<G/>` компонент з keyframes

## API-клієнт

```js
const API_BASE = import.meta.env.VITE_API_URL || "";
async function api(path, options = {}) { ... }
const apiGet  = p => api(p);
const apiPost = (p,b) => api(p, { method:"POST", body:b });
```

Авторизація: заголовок `X-Telegram-Init-Data: tg.initData`. Dev-режим: `X-Dev-User-Id: <id>`.

## Дизайн-токени

Єдина мова дизайну. **Ніколи не хардкодити кольори/відступи — тільки через токени.**

### C — Кольори
```js
C.bg      // #0a0a0a — основний фон
C.s1      // #111111 — картки
C.s2      // #161616 — вкладені елементи
C.s3      // #1c1c1c — третій рівень
C.acc     // #c8f53a — акцент (жовто-зелений)
C.acc2    // #a8d420 — темніший акцент
C.accDim  // rgba(200,245,58,0.12) — приглушений акцент
C.tm      // #ffffff — основний текст
C.ts      // #888888 — другорядний текст
C.td      // #444444 — приглушений текст
C.bc      // #222222 — межі
C.gradAcc // gradient акцент (для primary кнопок)
C.gradAccSubtle // тонкий акцентний градієнт (для accent Card)
```

### SP — Відступи (кратно 4px)
```js
SP[1]=4, SP[2]=8, SP[3]=12, SP[4]=16, SP[5]=20,
SP[6]=24, SP[8]=32, SP[10]=40, SP[12]=48
```

### R — Радіуси
```js
R.sm=8, R.md=12, R.lg=16, R.xl=20, R.full=9999
```

### F — Шрифтові стилі
```js
F.hero   // 34px 900wt — головний заголовок
F.h1     // 26px 900wt
F.h2     // 20px 800wt
F.h3     // 17px 800wt
F.body   // 14px 500wt
F.bodyLg // 15px 500wt
F.caption// 12px 600wt
F.meta   // 11px 700wt CAPS — мітки секцій
```

### SH — Тіні
```js
SH.sm, SH.md, SH.lg, SH.xl  // box-shadow з rgba(0,0,0,...)
SH.glow   // 0 0 24px rgba(200,245,58,0.25) — акцентне свічення
SH.inner  // inset 0 1px 0 rgba(255,255,255,0.04)
```

### T — Тривалості анімацій
```js
T.fast="150ms", T.base="220ms", T.slow="320ms"
```

### E — Easing-функції
```js
E.out    // cubic-bezier(0.16,1,0.3,1)  — spring
E.inOut  // cubic-bezier(0.65,0,0.35,1)
E.bounce // cubic-bezier(0.34,1.56,0.64,1)
```

## Тарифи — PLANS_STATIC (рядок 583)

```js
PLANS_STATIC = {
  start:   { price:799,  stars:1450, ... },
  premium: { price:1699, stars:3100, hot:true, ... },
  vip:     { price:3499, stars:6350, ... },
}
DUR_DISC = {1:0, 3:15, 6:25, 12:35}  // знижки %
```

**ПРАВИЛО:** Значення `price` і `stars` мусять збігатись з `PLANS`/`BASE_STARS` в `api.py` та `PLANS`/`STARS_PRICES` в `fitcore_v2.py`. Будь-яка зміна ціни — в трьох місцях одночасно.

## Важливі правила

- **Все в одному файлі.** `src/FitCoreApp.jsx` — не розбивати на окремі файли без явної потреби.
- **Inline styles + design tokens.** Не вводити CSS-файли або Tailwind — дотримуватись існуючого патерну.
- **Haptics.** Кожен клік → `haptic("light")`, підтвердження → `haptic("success")`.
- **Ціни** — не змінювати без синхронізації з backend.
- **Розташування:** `~/fitcore-miniapp` (перенесено з `~/Desktop/fitcore-miniapp`)
- **git:** окремий репо від `fitcore-bot` (`~/fitcore-bot`). Пушити незалежно.
- **Admin access:** визначається по `user.id === ADMIN_ID` у відповіді `/api/auth`.
